統合に関する詳細な手順から、決済フローを最適化するためのヒントまで、オンライン取引をスムーズに管理するために必要なすべての情報が揃っています。

安全で信頼性が高く、使いやすい決済環境をお客様に提供できるようサポートいたします！

## **支払いの受け入れ**

Payment Optionsは、柔軟かつ安全にオンライン決済を受け入れる方法を提供します。複数の統合方法から、ビジネスに最適なものを簡単に選択できます。

![オンライン決済](/images/flows-online-jp.svg)

* **ホスト決済ページ (HPP)**: 機密データを直接処理せずに、安全かつ迅速に支払いを受け入れる方法。

* **API**: より高度な統合を求めるビジネス向けに、購入および認可のためのAPIを提供し、支払いフローを完全に制御できます。

* **ウェブサイトプラグイン**: 人気のあるEコマースプラットフォーム向けに設計された、すぐに使用できるプラグインで、簡単に決済を開始できます。

## **ホスト型決済ページ (HPP)**

**ホスト型決済ページ**は、安全でカスタマイズ可能なチェックアウト体験を提供します。この方法では、顧客をホストされたページにリダイレクトし、そこで決済情報を入力できるため、PCI準拠とデータの安全性を確保できます。

![ホスト型決済ページ](/images/flows-hpp-jp.svg)

<Note> HPPリンクを使用して、顧客を安全なページへ誘導し、支払い情報を入力させることができます。この方法は、PCI準拠の負担を軽減したい企業に最適です。</Note>

## **オンライン決済のAPI**

支払いプロセスを完全に制御するには、APIを使用してウェブサイトやアプリケーションに直接統合してください。 **PURCHASE** および **AUTHORIZATION** APIを直接統合することで、取引の管理に柔軟性を持たせ、カスタム決済体験を構築できます。

![決済API](/images/flows-online-API-jp.svg)

### **PURCHASE API**

* **目的**: PURCHASE APIは、一回限りの支払いを開始するために使用されます。

* **機能**: カードやデジタルウォレットを含むさまざまな支払い方法をサポートします。

* **エンドポイント**: セットアップと使用ガイドラインについては、[Purchase API Documentation](/api-reference/en/server-to-server/purchase) を参照してください。

### **AUTHORIZATION API**

* **目的**: AUTHORIZATION APIは、指定された金額を即時に取得せずに保持するために使用され、事前認可に一般的に使用されます。

* **機能**: 資金を認可し、後で取得することができます。

* **エンドポイント**: 実装手順の詳細については、[Authorization API Documentation](/api-reference/en/server-to-server/authorization) をご確認ください。

<Tip> ヒント: 直接支払いにはPURCHASE APIを、ホテルの予約やレンタカーなどの資金保持にはAUTHORIZATION APIを使用してください。</Tip>
