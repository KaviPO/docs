加盟店は、複数のプランを設定し、カスタマイズ可能な請求間隔や料金設定を適用できます。また、メールやWhatsAppなどのチャネルを通じてサブスクリプションリンクを共有できます。

顧客は、安全な**ホスト型決済ページ（HPP）**リンクを通じて簡単にサブスクリプションに登録できます。このプロセスでは、プランの詳細を確認し、支払い情報を入力し、将来の支払いを自動化することが可能です。この効率的なアプローチにより、サブスクリプションの管理が容易になり、顧客の利便性と支払い状況を向上させます。定期収益モデルを持つ企業にとって理想的なソリューションです。

## **サブスクリプションの仕組み**

サブスクリプション製品では、複数のサブスクリプションプランを作成・管理でき、それぞれに異なる請求設定を適用できます。プランを作成したら、顧客にリンクを送信し、登録して支払い情報を入力してもらうことができます。

## **サブスクリプションのパラメータ**

以下は、サブスクリプションプランを設定およびカスタマイズするためのパラメータです：

```json
{
  "subscription": {
      "plan_name": "<string>",
      "plan_desc": "<string>",
      "cycle_type": "<DAYS/MONTHS>",
      "subscription_end_type": "CYCLE",
      "cycle_interval": <int>,
      "max_cycle_count": <int>,
      "has_trial_period": <boolean>,
      "trial_period_duration_type": "DAYS",
      "trial_period_duration": <int>,
      "has_discount": <boolean>,
      "discount_percentage": <int>,
      "discount_duration": <int>
  }
}
```
## **主要パラメータ**

サブスクリプションプランを設定するための重要なパラメータを紹介します：

* **`plan_name`**: サブスクリプションプランの名前。

* **`plan_desc`**: サブスクリプションプランの説明。

* **`cycle_type`**: 繰り返しの頻度（例: DAYS または MONTHS）。

* **`cycle_interval`**: 請求サイクルが繰り返される間隔を指定します。

* **`max_cycle_count`**: サブスクリプションの最大サイクル数。

* **`has_trial_period`**: 試用期間が含まれているかどうかを示します。

* **`trial_period_duration_type`**: 試用期間の単位（例: DAYS）。

* **`trial_period_duration`**: 試用期間の長さ。

* **`has_discount`**: 割引が適用されるかどうかを指定します。

* **`discount_percentage`**: サブスクリプション価格に適用される割引率。

* **`discount_duration`**: 割引が適用される期間の長さ。

## **定期支払いの設定方法**

<Steps>
  <Step title="DASGateway にログイン">
    * DASGateway にログインし、加盟店詳細画面を開きます。
  </Step>

  <Step title="サブスクリプション製品にアクセス">
    製品リストに移動し、追加された製品リストからサブスクリプション製品を選択します。
  </Step>

  <Step title="サブスクリプションプランを作成">
    サブスクリプション製品内で、新しいサブスクリプションプランを作成し、プラン名、請求頻度、支払い金額などの詳細を指定します。
  </Step>

  <Step title="サブスクリプションプランの管理">
    サブスクリプションプランを作成すると、利用可能なプラン一覧に表示されます。必要に応じて、プランを確認、更新、または複製できます。
  </Step>

  <Step title="サブスクリプションの開始 / サブスクライバーの追加">
    サブスクリプションを開始するには、プランを選択し、顧客に割り当てます。リンクの有効期限とプランの開始日を指定し、「送信」ボタンをクリックします。ユニークな登録リンクが生成されます。
  </Step>

  <Step title="サブスクリプションリンクの送信">
    生成された (HPP) リンクを顧客と共有し、プランの確認、支払い情報の入力、登録を完了できます。
  </Step>

  <Step title="サブスクライバー情報の管理">
    顧客がサブスクリプションに登録すると、サブスクライバ管理セクションに情報が保存されます。必要に応じて、情報の確認や更新が可能です。
  </Step>
</Steps>

## **アクティブなサブスクリプションの一時停止、キャンセル、再開**

アクティブなサブスクリプションの管理: 一時停止、キャンセル、再開

**1. サブスクリプションの一時停止**

**目的**: サブスクリプションを一時的に停止し、完全にキャンセルすることなく維持できます。

**手順**:
* サブスクライバー管理セクションをクリックします。
* 一時停止したいアクティブなサブスクリプションを選択します。
* 「一時停止」オプションをクリックし、操作を確認します。

**影響**:
* サブスクリプションが一時停止されている間は、請求が発生しません。
* サブスクリプションは、加盟店またはユーザーによっていつでも再開できます。

**2. サブスクリプションのキャンセル**

**目的**: サブスクリプションを完全に停止し、今後の請求を防ぎます。

**手順**:
* サブスクライバー管理セクションに移動します。
* キャンセルするサブスクリプションを選択します。
* 「キャンセル」ボタンをクリックし、操作を確認します。

**影響**:
* サブスクリプションは即時に終了するか、対応している場合は請求サイクルの終了時に終了します。
* キャンセルされたサブスクリプションのサービスは顧客に提供されなくなります。

**3. サブスクリプションの再開**

**目的**: 一時停止したサブスクリプションを再開し、請求とサービスを継続できます。

**手順**:
* サブスクライバー管理セクションで一時停止したサブスクリプションを見つけます。
* 「再開」オプションを選択し、操作を確認します。

**影響**:
* 元のサブスクリプション条件に従って請求が再開されます。
* 顧客に対するサービスが再開されます。

## **サブスクリプション管理の最適な運用方法**

* **明確なプラン名を作成**: サブスクリプションプランには管理しやすい明確で具体的な名前を付けてください。

* **適切な請求間隔を設定**: 顧客の期待に合った請求頻度と金額を設定し、解約率を低減します。

* **サブスクライバー情報を定期的に確認**: サブスクライバー情報を最新の状態に保ち、サブスクリプションの状況を定期的に見直します。
