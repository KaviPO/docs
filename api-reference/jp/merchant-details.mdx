## **目的**

**加盟店詳細ページ** は、APIペイロードを構築するための必要な設定情報を一元化し、加盟店が Payment Optionsゲートウェイとシームレスに連携できるようにします。これらの詳細にアクセスすることで、開発者は加盟店固有の設定に適合するペイロードを設定でき、互換性を向上させ、取引エラーを削減できます。

## **加盟店詳細ページで利用可能な主要情報**

加盟店詳細ページでは、適切なAPI設定に不可欠な以下の主要情報が提供されます。

### **加盟店ID**

* **加盟店ID** は、各加盟店に割り当てられた一意の識別子です。

* **APIペイロードに必須**: このIDはAPIリクエストに含める必要があり、Payment Optionsゲートウェイ内の加盟店アカウントを識別します。

* **一貫性**: 取引が正しい加盟店アカウントに紐付けられることで、正確な処理とレポート作成が可能になります。

### **処理通貨**

* 加盟店が取引を処理する際の通貨を定義します。

* **通貨サポート**: 処理通貨は、加盟店の地域および業務要件に適合している必要があります。

* **例**: JPY（日本円）、USD（米ドル）、PHP（フィリピンペソ）。

* **設定**: 適切な通貨を設定することで、顧客に正確な金額を請求でき、国際取引における通貨換算にも影響を与えます。

### **スキームタイプ**

* Visa や MasterCard など、加盟店アカウントに設定された決済スキーム。

* **スキーム設定**: 加盟店が受け入れるカードの種類を決定し、顧客の支払い方法の範囲に影響を与えます。

* **使用**: この設定は、加盟店のターゲット顧客と推奨カードネットワークに適合する必要があります。

## **取引タイプ**

加盟店詳細ページでは、決済の処理方法を決定する取引タイプも指定されます。

<CardGroup>
  <Card title="3DS（3Dセキュア）" icon="shield-check">
    オンライン取引向けのセキュリティプロトコルで、追加の顧客認証が必要です。セキュリティ強化のために推奨され、高リスクの取引における不正防止に役立ちます。
  </Card>

  <Card title="非3DS" icon="lock-open">
    3DS認証を必要としない標準の取引。低リスクの取引や3DSが必須でない場合によく使用されます。
  </Card>

  <Card title="定期支払い" icon="calendar">
    サブスクリプションなどのスケジュールされた自動取引。APIが事前設定されたスケジュールに基づいてこれらの取引をトリガーし、顧客の手動入力を不要にします。
  </Card>

  <Card title="トークン化" icon="key">
    生のカード情報の代わりに保存されたトークンを使用する安全な取引。PCI準拠を維持しながら、カード情報を保存した決済を可能にします。リピーターや会員向けに最適です。
  </Card>
</CardGroup>

## **その他の重要な情報**

上記の主要な詳細情報に加え、加盟店詳細ページには、業務運営に不可欠なその他の設定情報も含まれています。

### **加盟店の決済設定**

* 取引処理に影響を与える特定の決済設定の詳細。

* **設定オプション**: これには、デフォルトの決済時間、優先認証設定、その他の加盟店固有の取引設定が含まれる場合があります。

### **対応決済方法**

* **利用可能な決済方法**: ページには、加盟店が利用できるすべての決済方法が一覧表示されます。

  * **Apple Pay**

  * **Google Pay**

  * **GCash**

* **マルチチャネルサポート**: 複数の決済方法を提供することで、顧客が好きな支払い方法を選択でき、コンバージョン率が向上します。

### **制限と制約**

* 加盟店に適用される取引制限と処理制約を定義します。

* **取引制限**: 1回の取引で処理できる最大金額や1日の取引上限。

* **制約**: 地域制限、最大取引頻度、カードタイプの制限などが含まれ、地域の規制や業務ルールに準拠するために設定されます。

## **APIペイロード設定における加盟店詳細の活用**

これらの詳細情報にアクセスし、適切に実装することで、加盟店や開発者は正確かつ効率的なAPIペイロードを構築できます。

* **取引精度の向上**: すべての取引が正しい設定でルーティングされ、処理されることを保証します。

* **セキュリティとコンプライアンスの強化**: 3DSやトークン化などの取引タイプは、不正防止と規制基準の遵守を強化するために追加のセキュリティレイヤーを提供します。

* **カスタマイズ可能な顧客体験**: さまざまな決済方法をサポートし、加盟店固有の設定に適応することで、顧客体験が向上し、コンバージョン率の向上が期待できます。
